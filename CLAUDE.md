## Project overview

A VS Code extension providing an offline AI chat assistant powered by [Ollama](https://ollama.com). All inference runs locally ŌĆö no cloud, no API keys, no data leaks.

**Stack**: TypeScript 5, Node.js built-ins, VS Code extension API  
**No external NPM runtime dependencies** ŌĆö only `@types/*` and `typescript` in devDependencies.  
**Package**: `ollama-ai-assistant` v1.0.0 (published as `.vsix`)

## VS Code extension surface

**Activation**: `onStartupFinished`

**Commands** (5):
- `ollamaAI.openChat` ŌĆö open sidebar chat
- `ollamaAI.setup` ŌĆö open setup wizard
- `ollamaAI.explainCode` ŌĆö context menu (requires selection)
- `ollamaAI.fixCode` ŌĆö context menu (requires selection)
- `ollamaAI.generateDoc` ŌĆö context menu (requires selection)

**Sidebar**: `ollama-sidebar` view container ŌåÆ `ollamaAI.chatView` webview

**Configuration keys** (`ollamaAI.*`):
- `host` ŌĆö Ollama server URL (default `http://localhost:11434`)
- `model` ŌĆö active model ID (default `qwen2.5:3b`)
- `contextLines` ŌĆö lines of code sent as context (default `150`)
- `temperature` ŌĆö sampling temperature `0ŌĆō1` (default `0.3`)
- `streamResponse` ŌĆö enable token streaming (default `true`)

**Session storage keys** (`globalState`):
- `ollamaAI.chatSessions` ŌĆö map of `id ŌåÆ ChatSession`
- `ollamaAI.activeSessionId` ŌĆö currently active session ID
- `ollamaAI.setupDone` ŌĆö boolean; controls onboarding on first launch

## Key conventions

**Webview HTML** is embedded as template literals inside `_buildHtml()` methods. There is no external HTML/CSS bundler. Keep it self-contained; do not introduce webpack or esbuild without discussing the tradeoff.

**No external runtime dependencies.** The extension must remain zero-dependency at runtime. All network calls use Node.js built-ins (`http`, `https`).

**File edit safety** (`chatPanel.ts`): path validation rejects any path containing `..` before applying AI-generated edits to the workspace.

**Streaming abort** is handled via `AbortController` stored in `this._abortController`. Always cancel it before starting a new request.

**Session naming**: multilingual ŌĆö handles names starting with "Suhbat" (Uzbek for "Chat"). Keep this behavior when modifying `generateSessionName()`.

**Workspace file search** (`searchWorkspaceFiles`): excludes `node_modules`, `out`, `dist`, `.git`, `build`. Caps results at 400 (with query) / 80 (empty). Keep these limits when refactoring.

**Model library** (`systemDetector.ts` `ALL_MODELS`): filters by `ramGb >= ramRequired`. Adding a model here automatically surfaces it in onboarding recommendations.
